üìñ Overview

TaskFlow is a modern task management application built with React 19 and Vite. It provides a clean, intuitive interface for creating, organizing, and tracking tasks with features like filtering, sorting, dark mode, and local storage persistence.

The application follows a component-based architecture with custom hooks for state management and utility functions for business logic. All data is persisted in the browser's localStorage.

üéØ Main Component

src/App.jsx

Purpose: Main application component - orchestrates all features

State Management:

  • tasks: Array of task objects (persisted in localStorage via useLocalStorage)
  • isModalOpen: Controls TaskModal visibility
  • editingTask: Currently edited task (null for new tasks)
  • filters: Object containing searchQuery, category, priority, and status filters
  • theme: Current theme ('light' or 'dark') managed by useTheme hook

Key Functions:

  • handleCreateTask: Creates new task using createTask utility and adds to state
  • handleUpdateTask: Updates existing task using updateTask utility
  • handleDeleteTask: Removes task from state (with confirmation dialog)
  • handleToggleComplete: Toggles task completion status
  • handleOpenCreateModal: Opens modal for creating new task
  • handleOpenEditModal: Opens modal for editing existing task
  • handleSaveTask: Unified handler that calls create or update based on context

Computed Values (useMemo):

  • filteredAndSortedTasks: Applies filters and sorts by priority
  • stats: Calculates task statistics (total, active, completed, etc.)

Component Structure:

Header ‚Üí Search & Theme Toggle StatsBar ‚Üí Task Statistics FilterBar ‚Üí Category, Priority, Status Filters TaskList ‚Üí Grid of TaskCards Floating Action Button ‚Üí Create New Task TaskModal ‚Üí Create/Edit Form (conditionally rendered)

ü™ù Custom Hooks

src/hooks/useLocalStorage.js

Purpose: Synchronize React state with browser localStorage

  • API similar to useState: returns [value, setValue]
  • On mount: reads from localStorage or uses initial value
  • On update: automatically saves to localStorage whenever state changes
  • Handles JSON serialization/deserialization
  • Error handling for localStorage quota exceeded or disabled
  • Used for persisting tasks and theme preference

src/hooks/useTheme.js

Purpose: Manage application theme (light/dark mode)

  • Uses useLocalStorage to persist theme preference
  • Applies theme class to document.documentElement (HTML element)
  • Removes both 'light' and 'dark' classes before adding current theme
  • Provides toggleTheme function to switch between themes
  • Returns { theme, toggleTheme } object
  • Theme change triggers Tailwind's dark mode classes throughout the app

üõ†Ô∏è Utilities

src/utils/taskHelpers.js

Purpose: Business logic and data manipulation functions

Functions:

  • createTask(taskData):
    • Generates unique ID using UUID v4
    • Sets default values for missing fields
    • Adds createdAt and updatedAt timestamps
    • Returns complete task object
  • updateTask(task, updates):
    • Merges updates into existing task object
    • Updates updatedAt timestamp
    • Returns updated task
  • sortTasksByPriority(tasks):
    • Sorts by priority: High (3) ‚Üí Medium (2) ‚Üí Low (1)
    • Secondary sort: newest first (by createdAt)
    • Returns new sorted array (doesn't mutate original)
  • filterTasks(tasks, filters):
    • Filters by search query (title or description, case-insensitive)
    • Filters by category (if not 'all')
    • Filters by priority (if not 'all')
    • Filters by status (all/active/completed)
    • Returns filtered array
  • formatDate(dateString):
    • Formats ISO date string to human-readable format
    • Returns: "Today", "Tomorrow", "Yesterday", "X days ago", "In X days", or formatted date
    • Shows year only if different from current year
  • isTaskOverdue(task):
    • Checks if task due date has passed
    • Returns false if no due date or task is completed
    • Returns true if due date is in the past
  • getTaskStats(tasks):
    • Calculates statistics from task array
    • Returns object with: total, completed, active, overdue, highPriority counts

üîÑ Application Workflow

Initialization Flow:

  1. Browser loads index.html
  2. Vite loads main.jsx bundle
  3. React mounts App component to #root
  4. App.jsx initializes:
    • Loads tasks from localStorage (via useLocalStorage)
    • Loads theme preference from localStorage (via useTheme)
    • Applies theme class to HTML element
    • Initializes filter state
  5. Components render: Header, StatsBar, FilterBar, TaskList

User Interaction Flows:

Creating a Task:

  1. User clicks floating action button (FAB)
  2. handleOpenCreateModal sets editingTask = null and isModalOpen = true
  3. TaskModal renders with empty form
  4. User fills form and submits
  5. Form validation runs
  6. If valid: handleSaveTask ‚Üí handleCreateTask ‚Üí createTask utility
  7. New task added to state ‚Üí automatically saved to localStorage
  8. Modal closes, TaskList updates with new task

Editing a Task:

  1. User clicks Edit button on TaskCard
  2. handleOpenEditModal receives task object, sets editingTask and opens modal
  3. TaskModal populates form with task data
  4. User modifies and submits
  5. handleSaveTask ‚Üí handleUpdateTask ‚Üí updateTask utility
  6. Task updated in state ‚Üí saved to localStorage
  7. UI updates to show changes

Filtering Tasks:

  1. User changes filter in FilterBar (category, priority, or status)
  2. FilterBar calls setFilters with new value
  3. App's filter state updates
  4. filteredAndSortedTasks useMemo recalculates
  5. filterTasks utility filters task array
  6. sortTasksByPriority sorts filtered results
  7. TaskList re-renders with filtered/sorted tasks

Toggling Theme:

  1. User clicks theme toggle button in Header
  2. toggleTheme from useTheme hook executes
  3. Theme state toggles ('light' ‚Üî 'dark')
  4. Theme saved to localStorage
  5. useEffect in useTheme updates HTML class
  6. Tailwind dark mode classes activate throughout app
  7. All components re-render with new theme colors

üèóÔ∏è Architecture Summary

Technology Stack:

  • Framework: React 19 (with Hooks)
  • Build Tool: Vite (fast HMR, optimized builds)
  • Styling: Tailwind CSS (utility-first, dark mode support)
  • Icons: react-icons (Feather Icons)
  • ID Generation: uuid (v4)
  • Storage: Browser localStorage (via custom hook)
  • Code Quality: ESLint

Architecture Patterns:

  • Component-Based: Modular, reusable UI components
  • Custom Hooks: Reusable stateful logic (localStorage, theme)
  • Utility Functions: Pure functions for business logic
  • Constants: Centralized configuration data
  • Lifting State Up: App.jsx manages all shared state
  • Props Drilling: State and callbacks passed down via props

Data Flow:

  • Unidirectional: Data flows down (props), events flow up (callbacks)
  • State Management: React useState + useLocalStorage for persistence
  • Computed Values: useMemo for expensive calculations (filtering, stats)
  • Side Effects: useEffect for theme application and localStorage sync

Key Features:

  • ‚úÖ CRUD operations (Create, Read, Update, Delete tasks)
  • ‚úÖ Search functionality (title and description)
  • ‚úÖ Multi-filter system (category, priority, status)
  • ‚úÖ Priority-based sorting
  • ‚úÖ Task statistics dashboard
  • ‚úÖ Dark mode with persistence
  • ‚úÖ Responsive design (mobile, tablet, desktop)
  • ‚úÖ Form validation
  • ‚úÖ Overdue task detection
  • ‚úÖ Local storage persistence
  • ‚úÖ Smooth animations and transitions
